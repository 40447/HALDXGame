■ プロジェクト紹介

本プロジェクトで使用している技術スタックは以下の通りです。

レンダリング層（Rendering）： DirectX 11

物理エンジン： Jolt Physics

GUI： ImGui

アセット管理： 自作のリソース管理システム

ネットワーク通信： ENet を基盤とした自作ネットワークモジュール

モデル読み込み： Assimp（マルチフォーマット対応モデルインポートライブラリ）

■ プロジェクト構成

base： Gameplay 基盤フレームワーク

gameplay： ゲームロジック層

modules： 各種機能モジュール（ネットワーク、物理、エフェクトなど）

■ Gameplay フレームワーク紹介

Gameplay フレームワークは本プロジェクトの中核であり、
ゲーム全体の進行管理およびオブジェクトのライフサイクルを担っています。

全体設計は Unity の概念を参考にしており、以下の主要クラスで構成されています。

Game： ゲーム全体の初期化、メインループ、処理フローを管理するクラス

GameObject： ゲーム内の実体を表すオブジェクト

Component： GameObject に付加する機能単位のコンポーネント

■ プロジェクトの特徴
● 高いモジュール性

ネットワーク、物理などのモジュールはすべて Component ベースで実装され、
拡張性と保守性に優れています。

● マルチプレイ対応

ENet による高効率・低遅延のネットワーク通信を実現しています。

● 物理シミュレーション対応

Jolt Physics による物理挙動を使用し、ネットワーク同期にも対応しています。

● 組込みデバッグ UI

ImGui によるデバッグウィンドウを実装し、開発・検証が容易です。

● 多様な 3D モデルフォーマットに対応

Assimp を用いて複数形式の 3D モデル読み込みに対応しています。




專案介紹

本專案使用到的技術棧包含：
- 渲染層（Rendering）：DirectX 11
- 物理引擎：Jolt Physics
- 圖形化介面（GUI）：ImGui
- 資源管理：自研的資源管理系統
- 網路通訊：基於 ENet 的自製網路模組
- 模型載入：Assimp（多格式模型匯入庫）

專案結構
- base：Gameplay 基礎框架
- gameplay：遊戲邏輯層
- modules：功能模組（如網路、物理、特效等）

Gameplay 框架介紹

Gameplay 框架是整個專案的核心，負責管理遊戲的主要流程與物件生命週期。
整體架構參考 Unity 的設計概念，包含以下核心類別：
- Game：遊戲主類別，負責初始化、主迴圈與整體流程。

GameObject：遊戲物件，代表場景中的一個可運作實體。

Component：元件，掛載在 GameObject 上以提供特定功能。

專案特色

高度模組化設計
例如網路模組、物理模組等都是以 Component 形式實作，擴充與維護都很方便。

支援多人連線
使用 ENet 實現高效、低延遲的網路通訊。

真實物理效果
透過 Jolt Physics 提供物理模擬，並支援網路同步。

內建除錯介面
以 ImGui 建構開發時的除錯視窗，使調試與測試更直覺。

支援多種 3D 模型格式
使用 Assimp 匯入多種格式的 3D 模型。
